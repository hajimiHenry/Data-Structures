# 07 字符串

C 没有真正的「字符串类型」——字符串就是**以 `'\0'` 结尾的字符数组**。这一条是理解一切字符串行为的关键。

## 7.1 字符串 = 带 `\0` 的字符数组

```c
char s[] = "Hello";     // 实际存 H e l l o \0，占 6 字节！
char s[6] = "Hello";    // 刚好放下 \0
char s[5] = "Hello";    // ❌ 没空间放 \0，危险
```

- `"Hello"` 这种字符串字面量自带 `\0`，所以长度 5 的字符串占 6 字节。
- `\0` 是 ASCII 值 0 的字符，作为「字符串结束标记」。所有字符串函数靠它判断在哪结束。

```c
char s[20] = "Hi";
printf("%d\n", strlen(s));   // 2 —— 不含 \0
printf("%d\n", sizeof(s));   // 20 —— 数组总大小
```

`strlen`（内容长度）vs `sizeof`（数组占用），别混。

## 7.2 字符数组 vs 字符指针

```c
char a[] = "Hello";      // 数组：内容在 a 里，可改 a[0]
char *p = "Hello";       // 指针：p 指向只读的字面量，p[0]='h' 是未定义行为！
```

- 数组形式：拷贝一份到你的数组里，可修改。
- 指针形式：指向程序只读区的字面量，**只读**，改它会崩。
- 数据结构里字符串基本都用字符数组。

## 7.3 输入输出

```c
char s[100];
scanf("%99s", s);          // 读一个词（遇空格/换行停），不加 &，%99s 防溢出
printf("%s\n", s);         // 输出遇 \0 停

fgets(s, 100, stdin);      // 读一整行（含空格），更安全，末尾会带 \n
```

读整行带空格用 `fgets`；读单词用 `scanf %s`。

## 7.4 字符串函数（`<string.h>`）

| 函数 | 作用 | 注意 |
|------|------|------|
| `strlen(s)` | 长度，不含 `\0` | 不算结尾 |
| `strcpy(dst, src)` | 复制 | dst 要够大 |
| `strcat(dst, src)` | 拼接到末尾 | dst 要够大，从 `\0` 处接 |
| `strcmp(a, b)` | 比较 | 返回 `<0/0/>0` |
| `strchr(s, ch)` | 找字符 | 返回指针或 NULL |

```c
char dst[20];
strcpy(dst, "Hello");
strcat(dst, " World");        // dst = "Hello World"
if (strcmp(dst, "Hello World") == 0)   // 比较字符串必须用 strcmp
    puts("equal");
```

**最大坑**：比较字符串**不能 `==`**。`if (s == "abc")` 比的是地址（两个不同地方的 "abc" 地址不同），几乎永远不相等。必须 `strcmp(s, "abc") == 0`。

```c
char a[] = "abc";
char b[] = "abc";
if (a == b) ...       // ❌ 比地址，几乎肯定不相等
if (strcmp(a, b) == 0) ...   // ✅ 比内容
```

## 7.5 字符串作为函数参数

字符串就是 `char` 数组，传参同样退化为 `char *`：

```c
int my_strlen(const char *s) {
    int n = 0;
    while (*s) { n++; s++; }   // 遇 \0（即 0）停
    return n;
}
```

`const char *` 表示「指向只读字符的指针」——函数不打算改它，加 `const` 更安全。

## 7.6 字符串与结构体

结构体里放字符串成员用字符数组：

```c
typedef struct {
    char name[20];
    int  age;
} Person;

Person p;
strcpy(p.name, "Alice");   // 不能 p.name = "Alice"
p.age = 20;
```

字符串成员赋值必须 `strcpy`，不能用 `=`（数组不能赋值）。

## 易错点

1. `"abc"` 占 4 字节（含 `\0`），数组要开够。
2. `strlen`（长度）vs `sizeof`（占空间）。
3. 字符串比较用 `strcmp`，绝不用 `==`。
4. `char *p = "abc"; p[0]='A';` 未定义行为，改只读字面量会崩。
5. 字符串成员赋值用 `strcpy`，不能用 `=`。
6. `scanf %s` 不限长会溢出，用 `%99s`（比数组大小少 1 留给 `\0`）。

## 例题

**例 7.1** 手写 `strlen`。
```c
size_t my_strlen(const char *s) {
    size_t n = 0;
    while (s[n]) n++;
    return n;
}
```

**例 7.2** 判断回文串（你做过）。
```c
int is_palindrome(const char *s) {
    int i = 0, j = strlen(s) - 1;
    while (i < j) {
        if (s[i] != s[j]) return 0;
        i++; j--;
    }
    return 1;
}
```

**例 7.3** 下面哪里错？
```c
char *s;
strcpy(s, "hello");
```
答：`s` 是野指针，没指向有效内存，`strcpy` 往随机地址写，崩。要么用数组 `char s[20];`，要么 `s = malloc(20);`（08 章）。**这正是动态内存的典型场景**。

**例 7.4** 下面输出？
```c
char s[] = "abc";
char *p = s;
p[0] = 'A';
printf("%s\n", s);
```
答：`Abc`。p 和 s 指向同一块内存，改 p[0] 就是改 s[0]。

## 小练习

1. 写 `void reverse_str(char *s)`，原地反转字符串（如 "abc" → "cba"）。
2. 写 `int my_strcmp(const char *a, const char *b)`，模仿 `strcmp`。
3. 写 `int count_char(const char *s, char c)`，统计字符 c 在 s 中出现次数。

<details><summary>答案</summary>

```c
// 1
void reverse_str(char *s) {
    int i = 0, j = strlen(s) - 1;
    while (i < j) {
        char t = s[i]; s[i] = s[j]; s[j] = t;
        i++; j--;
    }
}

// 2
int my_strcmp(const char *a, const char *b) {
    while (*a && *a == *b) { a++; b++; }
    return (unsigned char)*a - (unsigned char)*b;
}

// 3
int count_char(const char *s, char c) {
    int n = 0;
    while (*s) { if (*s == c) n++; s++; }
    return n;
}
```
