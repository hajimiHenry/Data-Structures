# 04 数组

数组 = 一串**同类型**、**连续存放**的变量，共用一个名字，用下标区分。数据结构里顺序表、栈、队列底层都是数组。

## 4.1 一维数组

```c
int a[5] = {10, 20, 30, 40, 50};   // 长度 5，下标 0~4
// a[0]=10, a[1]=20, ..., a[4]=50
a[2] = 99;                          // 修改
```

- **下标从 0 开始**，到 `n-1`。`a[5]`（长度 5 时）越界。
- 部分初始化：`int a[5] = {1,2};` 剩余自动补 0。`int a[5] = {0};` 全 0。
- 不给长度让编译器数：`int a[] = {1,2,3};` → 长度 3。
- C **不检查越界**，`a[-1]`、`a[100]` 编译能过、运行可能崩或悄悄改坏别的变量——靠你脑子。

## 4.2 数组与循环

```c
int a[100];
int n; scanf("%d", &n);
for (int i = 0; i < n; i++) scanf("%d", &a[i]);   // 读 n 个
for (int i = n - 1; i >= 0; i--) printf("%d ", a[i]);  // 逆序输出
```

## 4.3 数组作函数参数：退化（重点）

把数组传给函数时，**数组名退化成「指向首元素的指针」**，长度信息丢失。

```c
int sum(int a[], int n) {     // a[] 退化为 int *a，长度 n 必须单独传
    int s = 0;
    for (int i = 0; i < n; i++) s += a[i];
    return s;
}
int main(void) {
    int x[5] = {1,2,3,4,5};
    printf("%d\n", sum(x, 5));   // 15
}
```

关键三点：
1. 形参 `int a[]` 实际是 `int *a`，**函数内拿不到数组长度**——必须额外传 `n`。
2. 在函数内 `sizeof(a)` 得到的是**指针大小**（8 字节），不是数组总大小。所以**别在函数里 `sizeof(a)/sizeof(a[0])` 算长度**。
3. 因为传的是首地址，**函数内改 `a[i]` 会改到原数组**（和普通变量值传递不同——这里传的就是地址）。所以数组天然能被函数修改，不用额外传指针。

## 4.4 二维数组

```c
int m[3][4];   // 3 行 4 列，共 12 个 int，连续存放
m[1][2] = 5;   // 第 1 行第 2 列
```

行优先存储：`m[0][0], m[0][1], ..., m[0][3], m[1][0], ...`。

### 关键认知：二维在内存里本质是一维

所谓"二维"是编译器脑里的概念——它把一长条连续的 12 个 int，**按每 4 个一组切成 3 行**。"行"和"列"靠列数维持，内存里全是平的：

```
内存：  0    1    2    3  | 4    5    6    7  | 8 ...
       第0行            第1行            第2行
```

> 「跨一整行」= 跳过 `cols` 个元素。列数是 4，从第 0 行第 0 个跨到第 1 行第 0 个，下标就 `+4`。
> 访问 `m[i][j]` = 起点 + `i*cols + j`（跨 `i` 行，每行 `cols` 个，再在这行走 `j` 个）。

### 二维数组传参：退化丢列数，靠调用方补

退化成 `int*` 后，起点、总长度、行数、**列数全丢**。而 `m[12]`、`m[3][4]`、`m[2][6]` 在内存里都是"12 个连续 int"，`int*` 只记起点不记形状——**一维二维已经无法区分**。补救靠两种写法：

```c
// 写法 A：列数写进类型，编译器替你算「加 4 跨行」
void print(int m[][4], int rows) {     // 等价 int (*m)[4]
    m[1][2] = 5;     // ✅ 能直接用 m[i][j]，编译器自动算成 起点+1*4+2
}
// ⚠ 列数被焊死成 4，传列数是 5 的数组就错

// 写法 B：完全退化成 int*，列数靠参数传，下标自己手算
void print(int *m, int rows, int cols) {
    m[1*cols + 2];   // ❌ 不能写 m[1][2]，要手动 m[i*cols+j]
}
int main(void) {
    int a[3][4];
    print(&a[0][0], 3, 4);   // 调用方负责告诉形状
}
```

「加 4 跨一行」这个动作**永远在发生**，区别只是：

| 写法 | 退化成 | 列数从哪来 | 谁算下标 | 能否 `m[i][j]` |
|------|--------|-----------|---------|---------------|
| `int m[][4]` | `int (*m)[4]` | 焊在类型（固定 4） | 编译器算 | ✅ |
| `int *m` | `int*` | 单独传 `cols` 参数 | 你手算 `i*cols+j` | ❌ |

> **核心**：二维数组的「维」靠列数维持。退化丢了列数，「维」就消失，必须由调用方重新告诉函数形状。想通这条，二维传参没有死角。

## 易错点

1. 下标越界——C 不检查，最隐蔽的 bug。
2. 数组传参后 `sizeof` 退化，不能在函数里算长度。
3. 二维形参忘写列数：`int m[][]` 非法，必须 `int m[][4]`。
4. `int a[n]` 用变量作长度是 C99 变长数组(VLA)，能用但别依赖，链表时代靠 malloc。
5. 数组之间不能直接赋值：`a = b;` 非法（数组名不可改），要循环拷贝。

## 例题

**例 4.1** 求数组最大值及其下标。
```c
int max = a[0], idx = 0;
for (int i = 1; i < n; i++)
    if (a[i] > max) { max = a[i]; idx = i; }
```

**例 4.2** 冒泡排序（你写过，回顾）。
```c
void bubble(int a[], int n) {
    for (int i = 0; i < n - 1; i++)
        for (int j = 0; j < n - 1 - i; j++)
            if (a[j] > a[j+1]) {
                int t = a[j]; a[j] = a[j+1]; a[j+1] = t;
            }
}
```

**例 4.3** 有序插入（你 study 文件里的题）：在有序数组中插入 x 保持有序。
```c
int i = n - 1;
while (i >= 0 && a[i] > x) { a[i+1] = a[i]; i--; }   // 比它大的右移
a[i+1] = x;
n++;
```

**例 4.4** 下面输出？
```c
void f(int a[], int n) { a[0] = 999; }
int main(void) {
    int x[3] = {1,2,3};
    f(x, 3);
    printf("%d\n", x[0]);   // 999，数组传的是地址，函数内修改影响原数组
}
```

## 小练习

1. 写 `int find(int a[], int n, int key)`，返回 key 在数组中第一次出现的下标，没有返回 -1。
2. 写 `void reverse(int a[], int n)`，原地反转数组。
3. 统计一个 int 数组中每个数字 0~9 出现次数（用计数数组）。

<details><summary>答案</summary>

```c
// 1 顺序查找
int find(int a[], int n, int key) {
    for (int i = 0; i < n; i++)
        if (a[i] == key) return i;
    return -1;
}

// 2 双指针反转
void reverse(int a[], int n) {
    for (int i = 0, j = n - 1; i < j; i++, j--) {
        int t = a[i]; a[i] = a[j]; a[j] = t;
    }
}

// 3 计数数组
int cnt[10] = {0};
for (int i = 0; i < n; i++) {
    int v = a[i];
    if (v >= 0 && v <= 9) cnt[v]++;   // 防越界
}
// cnt[d] 即数字 d 的出现次数
```
