# 08 动态内存管理 ⭐

**从这章开始是你完全没学过的部分，也是数据结构最依赖的部分。** 链表、树、图的节点都是运行时才知道要多少个，静态数组搞不定，必须动态分配。

## 8.1 为什么需要动态内存

回顾两个限制：

1. **数组大小必须编译时定死**：`int a[100]` 只能存 100 个。存多少运行时才知道怎么办？
2. **函数返回的局部变量/数组会失效**：想在函数里造一个东西返回出去给别人长期用，局部变量做不到（03 章、07 章例题都踩过）。

`malloc` 解决这两个问题：**在程序运行时，向系统申请一块内存，这块内存的生命周期由你控制——直到你主动释放，它一直有效**。

## 8.2 malloc / free 基本用法

```c
#include <stdlib.h>   // malloc, free, exit 都在这里

int *p = (int *)malloc(sizeof(int));   // 申请 1 个 int 的内存
if (p == NULL) {                       // 务必检查是否成功
    printf("内存不足\n");
    exit(1);
}
*p = 42;                               // 用这块内存
printf("%d\n", *p);
free(p);                              // 用完释放！
p = NULL;                             // 好习惯：释放后置空，防悬空
```

逐行：
- `malloc(size)`：向系统申请 `size` 字节内存，返回这块内存首地址（`void *`，即「无类型指针」）。
- `(int *)`：把 `void *` 强转成 `int *`（C 里可省，但写出来清晰，C++ 必须写）。
- **返回可能 NULL**（系统没内存了），每次都要检查。
- `free(p)`：把 `p` 指向的内存还给系统。**之后 p 自身没变（还存着那个地址），但那块内存已不属于你**——再用就是悬空指针。所以 free 后立即 `p = NULL;`。
- **配对铁律**：每个 `malloc` 必须有且只有一个 `free`。少 free = 内存泄漏；多 free / 重复 free = 崩溃。

## 8.3 申请多个元素：动态数组

`malloc` 最常见的用法是申请一整片，当数组用：

```c
int n;
scanf("%d", &n);

int *a = (int *)malloc(n * sizeof(int));   // 申请 n 个 int，运行时才知道 n
if (a == NULL) exit(1);

for (int i = 0; i < n; i++) a[i] = i;       // 当数组用，a[i] 完全合法
for (int i = 0; i < n; i++) printf("%d ", a[i]);

free(a);
a = NULL;
```

`malloc` 出来的内存，用 `a[i]` 或 `*(a+i)` 访问都行，和数组没区别。这就是**动态数组**——大小运行时定。

> `n * sizeof(int)` 而不是 `n * 4`：用 `sizeof` 跨平台、跨类型不会错。

## 8.4 calloc / realloc

**calloc**：申请并自动清零（malloc 出来的内存是随机垃圾值）。
```c
int *a = (int *)calloc(n, sizeof(int));   // n 个 int，全 0
// 用法：calloc(元素个数, 每个大小)
free(a);
```

**realloc**：改变已分配内存的大小（扩容/缩容）。
```c
int *a = malloc(10 * sizeof(int));
// ... 装满了，要扩容到 20
int *tmp = realloc(a, 20 * sizeof(int));   // 可能返回新地址
if (tmp == NULL) { /* 扩容失败，a 仍可用，自己处理 */ }
else { a = tmp; }                           // 成功，更新指针
```

> realloc 三种情况：原地扩容（返回原地址）；搬到新地方（返回新地址，旧内容自动拷过去，旧内存自动释放）；失败返回 NULL（原内存仍有效）。**所以用临时变量接 realloc 结果**，成功后再赋回原指针，避免失败时丢失原内存。

## 8.5 动态内存的生命周期（核心概念）

```c
int *make_array(int n) {
    int *a = malloc(n * sizeof(int));   // 堆上分配
    for (int i = 0; i < n; i++) a[i] = i;
    return a;     // ✅ 合法！堆内存不会随函数结束失效
}

int main(void) {
    int *p = make_array(5);
    printf("%d\n", p[3]);   // 3，能用
    free(p);               // 谁拿到谁负责释放
    return 0;
}
```

对比局部数组 `int a[5]; return a;` ❌（出函数失效）。**堆内存（malloc 出来的）的生命周期由你 free 决定，不受函数作用域限制**——这就是链表能「函数里造节点、串起来、返回出去」的根本原因。

## 8.6 四大经典错误

1. **内存泄漏**（最常见）：malloc 了不 free。短程序没事，长期运行程序（服务器）会耗尽内存。**养成「谁申请谁释放」习惯**。

2. **悬空指针**：free 后还在用。
   ```c
   free(p); *p = 5;   // ❌ p 已释放，行为未定义
   ```

3. **重复释放**：对同一地址 free 两次，崩。
   ```c
   free(p); free(p);   // ❌
   ```

4. **越界写**：申请了 5 个，写了第 6 个。
   ```c
   int *a = malloc(5 * sizeof(int));
   a[5] = 0;   // ❌ 越界，可能悄悄改坏别的内存
   ```

> 用 `-fsanitize=address` 编译能立刻抓住 2、3、4 这些错误，学这章时强烈建议开着：
> ```bash
> clang -Wall -fsanitize=address a.c -o a && ./a
> ```

## 8.7 堆 vs 栈（理解原理）

- **栈**：局部变量、函数参数，自动分配自动回收，速度快但大小有限（通常几 MB）。函数返回即失效。
- **堆**：malloc 出来的，手动管理，大小大（GB 级），生命周期你定。
- 数据结构的节点都放堆上（因为数量、生命周期运行时才知道）。

## 易错点

1. malloc 后不检查 NULL。
2. malloc 的 `sizeof` 写错类型，或漏 `*` 导致大小不够。
3. 忘 free → 泄漏；重复 free → 崩。
4. free 后还用指针 → 悬空。养成 `free(p); p = NULL;`。
5. realloc 不用临时变量接，失败时丢失原指针。
6. 越界访问动态数组。

## 例题

**例 8.1** 动态数组：读 n，申请 n 个 int，读入，输出最大值，释放。
```c
#include <stdio.h>
#include <stdlib.h>
int main(void) {
    int n; scanf("%d", &n);
    int *a = malloc(n * sizeof(int));
    if (!a) return 1;
    for (int i = 0; i < n; i++) scanf("%d", &a[i]);
    int max = a[0];
    for (int i = 1; i < n; i++) if (a[i] > max) max = a[i];
    printf("%d\n", max);
    free(a);
    return 0;
}
```

**例 8.2** 动态字符串：函数返回一个新分配的字符串拷贝。
```c
char *dup_str(const char *s) {
    char *p = malloc(strlen(s) + 1);   // +1 给 \0！
    if (!p) return NULL;
    strcpy(p, s);                       // 拷贝含 \0
    return p;
}
int main(void) {
    char *copy = dup_str("Hello");
    printf("%s\n", copy);
    free(copy);                         // 谁拿到谁释放
}
```
关键：**字符串要 `strlen + 1` 留给 `\0`**，少 1 字节经典 bug。

**例 8.3** 下面哪里泄漏？
```c
void f(void) {
    int *p = malloc(100 * sizeof(int));
    p = malloc(50 * sizeof(int));   // ❌ 第一个 100 块丢了，泄漏
    free(p);                        // 只释放了第二个
}
```
答：第一次 malloc 的内存地址被第二次覆盖，再也无法 free → 泄漏。应先 free 第一个再重新分配，或用 realloc。

## 小练习

1. 动态数组扩容：初始申请 4 个 int，读入整数直到 EOF；装满就 realloc 翻倍扩容，最后输出所有数。
2. 写 `int *range(int n)`，返回动态分配的数组 `{0,1,2,...,n-1}`，调用方负责释放。
3. 找下面程序的内存错误：
```c
char *get_name(void) {
    char buf[10] = "Alice";
    return buf;
}
```

<details><summary>答案</summary>

```c
// 1 动态扩容（vector 思路）
int cap = 4, n = 0;
int *a = malloc(cap * sizeof(int));
int x;
while (scanf("%d", &x) == 1) {
    if (n == cap) {                 // 装满
        cap *= 2;
        int *tmp = realloc(a, cap * sizeof(int));
        if (!tmp) { free(a); return 1; }
        a = tmp;
    }
    a[n++] = x;
}
for (int i = 0; i < n; i++) printf("%d ", a[i]);
free(a);

// 2
int *range(int n) {
    int *a = malloc(n * sizeof(int));
    if (!a) return NULL;
    for (int i = 0; i < n; i++) a[i] = i;
    return a;
}
// 调用：int *r = range(5); ...; free(r);

// 3
// buf 是局部数组（栈），出函数失效，返回的是悬空指针。
// 修正：char *p = malloc(10); strcpy(p, "Alice"); return p;
//      （调用方负责 free）—— 这正是动态内存的核心用途。
```
练习 1 的「装满翻倍扩容」是顺序表动态扩容的标准做法，408 常考，吃透它。
