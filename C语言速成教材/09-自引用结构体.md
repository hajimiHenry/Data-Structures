# 09 自引用结构体 ⭐

这章是连接「结构体+指针+动态内存」三者和「链表/树」的桥梁。学完它，链表节点对你就是顺理成章。

## 9.1 问题：怎么表示「一串数据，但个数运行时才知」

数组做不到（要编译时定大小）。一个自然的想法：每个数据装进一个结构体小盒子，盒子里再放一个**指向下一个盒子的地址**——像寻宝游戏「线索1 → 线索2 → 线索3 → 终点」。这就是链表。

## 9.2 自引用结构体：结构体里含「指向自己的指针」

```c
typedef struct Node {
    int data;              // 数据
    struct Node *next;     // 指向「下一个同类节点」的指针
} Node;
```

**关键点**：
- `struct Node *next` 是指针，不是 `struct Node next`。如果是后者（结构体里含一个完整的自己），大小无穷递归，编译器报错。**指针大小固定（8 字节），不递归**——这就是为什么必须用指针。
- 成员 `next` 的类型是 `struct Node *`——「指向自身类型的指针」，这就是「**自引用结构体**」。

> 注意 typedef 顺序：在 `typedef struct Node { ... } Node;` 内部，类型名 `Node` 还没定义完，所以内部必须用 `struct Node *next`，不能用 `Node *next`。这是 C 的硬性规则，照着写。

## 9.3 一个节点长什么样

```c
Node *node = (Node *)malloc(sizeof(Node));   // 堆上造一个节点
node->data = 42;
node->next = NULL;     // 先让它不指向任何地方
```

内存里：
```
node ──► [ data=42 | next=NULL ]
```

`node` 是指针，指向堆上一个 `Node`。`node->data` 是数据，`node->next` 是「下一个节点的地址」（现在是 NULL，表示没有下一个）。

## 9.4 把两个节点串起来：链表雏形

```c
Node *a = malloc(sizeof(Node));   a->data = 1;
Node *b = malloc(sizeof(Node));   b->data = 2;

a->next = b;        // a 指向 b
b->next = NULL;     // b 是最后一个

// 现在从 a 出发能走到 b
Node *head = a;     // head 是链表「头指针」，唯一入口
```

内存里：
```
head ──► [1|*]──► [2|NULL]
        a          b
```

**链表的灵魂**：只要抓住 `head`，顺着每个节点的 `next` 就能走到所有节点。每个节点本身没有名字，靠前一个节点的 `next` 找到它。

## 9.5 遍历链表

```c
void print_list(Node *head) {
    Node *cur = head;          // 用一个游标从头走，别动 head 本身
    while (cur != NULL) {      // 走到 NULL 停
        printf("%d ", cur->data);
        cur = cur->next;       // 走到下一个
    }
    printf("\n");
}
```

**遍历三步曲**（背下来，所有链表操作的基础）：
1. 用一个游标指针从 `head` 出发（不动 head）。
2. `while (cur != NULL)` 循环。
3. 每轮 `cur = cur->next` 往前走。

## 9.6 链表 vs 数组（理解 trade-off）

| | 数组 | 链表 |
|---|---|---|
| 内存 | 连续 | 分散（靠指针连） |
| 随机访问 `a[i]` | O(1) | O(n)（要遍历） |
| 头部插入/删除 | O(n)（要挪） | O(1)（改指针即可） |
| 大小 | 编译时/ realloc | 运行时随节点数变 |
| 额外开销 | 无 | 每个节点多一个指针 |

数据结构选哪个看操作：**频繁头部增删用链表，频繁随机访问用数组**。这就是为什么 408 顺序表和链表都学。

## 9.7 释放整条链表

malloc 的节点必须逐个 free。**不能只 free 头**——后面的节点全泄漏。

```c
void free_list(Node *head) {
    Node *cur = head;
    while (cur != NULL) {
        Node *next = cur->next;   // 先存好下一个的地址！
        free(cur);                // 再释放当前
        cur = next;               // 走到下一个
    }
}
```

**关键易错**：必须先用临时变量存 `cur->next`，再 `free(cur)`。如果先 free 再取 `cur->next`，cur 已释放，`cur->next` 是未定义行为。

## 易错点

1. 结构体里写 `Node next`（应为 `Node *next` / `struct Node *next`）——递归无限大。
2. typedef 内部用别名（应用 `struct Node *next`）。
3. 遍历时直接 `head = head->next` 破坏头指针——用游标 `cur`。
4. free 链表时先 free 再取 next——悬空。
5. 忘记把新节点的 `next` 初始化（NULL 或指向某节点）——野指针。
6. `node->data` 当 node 为 NULL 时崩——访问前判空。

## 例题

**例 9.1** 手工建 3 个节点的链表 1→2→3 并遍历。
```c
#include <stdio.h>
#include <stdlib.h>

typedef struct Node { int data; struct Node *next; } Node;

int main(void) {
    Node *n1 = malloc(sizeof(Node));  n1->data = 1;
    Node *n2 = malloc(sizeof(Node));  n2->data = 2;
    Node *n3 = malloc(sizeof(Node));  n3->data = 3;
    n1->next = n2; n2->next = n3; n3->next = NULL;

    Node *head = n1;
    Node *cur = head;
    while (cur) { printf("%d ", cur->data); cur = cur->next; }
    printf("\n");   // 1 2 3

    // 释放
    cur = head;
    while (cur) { Node *nx = cur->next; free(cur); cur = nx; }
    return 0;
}
```

**例 9.2** 求链表长度。
```c
int length(Node *head) {
    int n = 0;
    for (Node *cur = head; cur; cur = cur->next) n++;
    return n;
}
```

**例 9.3** 下面哪里错？
```c
typedef struct { int data; Node *next; } Node;
```
答：typedef 块内 `Node` 还没定义就用，且结构体内部需 `struct Node *`。正确：
```c
typedef struct Node { int data; struct Node *next; } Node;
```

**例 9.4** 数数：下面链表遍历会打印什么？
```c
Node *head = malloc(sizeof(Node)); head->data = 5; head->next = NULL;
Node *cur = head;
while (cur->next != NULL) {            // 注意是 cur->next != NULL
    printf("%d ", cur->data);
    cur = cur->next;
}
```
答：什么都不打印。因为 head 的 next 就是 NULL，循环条件一开始就假。**遍历通常用 `while (cur != NULL)`**；`while (cur->next != NULL)` 适用于「停在最后一个节点上」的场景（如尾插），区别要清楚。

## 小练习

1. 写 `Node *make_node(int data)`：malloc 一个节点，填 data，next 置 NULL，返回指针。
2. 写 `int sum_list(Node *head)`，求链表所有 data 之和。
3. 写 `Node *find(Node *head, int target)`，返回第一个 data==target 的节点指针，没有返回 NULL。
4. 写 `void free_list(Node *head)`，正确释放整条链表。

<details><summary>答案</summary>

```c
// 1
Node *make_node(int data) {
    Node *n = malloc(sizeof(Node));
    if (!n) return NULL;
    n->data = data;
    n->next = NULL;
    return n;
}

// 2
int sum_list(Node *head) {
    int s = 0;
    for (Node *cur = head; cur; cur = cur->next) s += cur->data;
    return s;
}

// 3
Node *find(Node *head, int target) {
    for (Node *cur = head; cur; cur = cur->next)
        if (cur->data == target) return cur;
    return NULL;
}

// 4
void free_list(Node *head) {
    Node *cur = head;
    while (cur) {
        Node *nx = cur->next;   // 先存
        free(cur);             // 再释放
        cur = nx;
    }
}
```
