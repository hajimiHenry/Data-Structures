# 05 指针

**这是数据结构的地基**。链表、树全是「一个东西里放着指向另一个东西的地址」。这章从零讲，慢一点，**每个例子都自己敲一遍跑一遍**。指针不难，只是被讲玄了——它就是个「存地址的变量」。

---

## 5.1 先理解「地址」

内存是一长排格子，每格有编号（地址）。你声明 `int x = 42;` 时，编译器在内存里挑几个格子存 42，并记住它的起始地址，比如地址是 `0x1000`：

```
地址:   ... 0x1000  0x1004  0x1008 ...
内容:   ... [  42  ] [      ] [      ] ...
              ↑
              x 住在这里（int 占 4 字节，占 0x1000~0x1003）
```

平时你用名字 `x` 访问它。`&x` 就是「取出 x 的地址」——`0x1000`。

```c
int x = 42;
printf("%p\n", &x);   // 打印地址，类似 0x7ffee...  每次运行不一样
printf("%d\n", x);    // 打印值：42
```

- `%p` 打印地址（指针）。
- **值** vs **地址**：`x` 是值 42，`&x` 是地址 0x1000。这俩是不同概念，分清是学指针的第一步。

## 5.2 指针：存地址的变量

指针就是普通变量，只是它存的内容是**另一个变量的地址**。

```c
int x = 42;
int *p = &x;     // p 是「指向 int 的指针」，里面存着 x 的地址 0x1000
```

内存图：

```
地址:   ... 0x1000 ...            ... 0x2000 ...
内容:   ... [ 42 ] ...            ... [0x1000] ...
              ↑                          ↑
              x                         p（存的是 x 的地址）
```

`p` 自己也住在某个地址（0x2000），但它的**值**是 42 这格的地址（0x1000）。**指针 = 指着别人的变量。**

### `int *p` 里的 `int` 描述的是「终点」，不是 `p` 自己

初学最易混：以为 `int *p` 是"p 是指针 + p 是整型"两个标签叠一起。**错。**

- `p` 自己永远只是个存地址的盒子（8 字节），跟 `int` 无关。
- 那个 `int` 是给**编译器**看的标签，意思是"**p 指向的那个终点是 int 类型**"。

```
int *p
└┬┘└┘
 │ └─ p 是个指针（存地址）
 └─── 标签：终点放的是 int 型货物
```

**类型不能瞎填，填错了取错数据甚至报错：**

| 表达式 | 意思 | 大小 |
|--------|------|------|
| `p` | 指针自己（存地址） | 8 字节，跟类型无关 |
| `*p` / 指向的东西 | 终点数据 | 由那个类型决定（int=4，double=8） |

验证：
```c
int x = 5; int *p = &x;
printf("%zu\n", sizeof(p));   // 8  ← p 自己是指针大小，跟 int 无关
printf("%zu\n", sizeof(*p));  // 4  ← 终点数据是 int，这才是那个 int 管的
```

> **一句话**：星号前的类型 = 终点的类型，一一对应。指向 `int` 写 `int *`，指向 `struct Node` 写 `struct Node *`（终点是结构体才能用 `->` 取成员），指向"指向 int 的指针"写 `int **`。换不同标签，同一个地址取出来的东西、能用的操作都不一样。

### `*` 的两个含义（别混）

`*` 在 C 里出现两种地方，含义不同：

1. **声明里** `int *p;` —— 这里的 `*` 是**类型标记**，表示「p 是指针类型」。不是运算。
2. **表达式里** `*p` —— 这叫**解引用**（dereference），表示「去 p 存的那个地址，把东西拿出来」。

```c
int x = 42;
int *p = &x;     // 声明：p 是指针，初始化为 x 的地址

printf("%d\n", *p);   // 解引用：跟着 p 存的地址走过去，拿到 42
*p = 100;             // 解引用并赋值：跟着地址走过去，把那格改成 100
printf("%d\n", x);    // 100 —— 改的就是 x！
```

**关键**：`*p` 不是「p 的值」，而是「p 指向的那格的值」。`p` 的值是地址，`*p` 才是那个地址处的数据。

### 两个运算符，互为逆操作

```
&p   ：取 p 的地址（把变量变成地址）
*p   ：解引用 p（把地址变回它指向的变量）
```

它们互逆：`*&x == x`，`&*p == p`。**记住这对关系，指针的语法就通了。**

```c
int x = 42;
int *p = &x;      // p ← &x（把 x 的地址给 p）

*p;               // 跟着 p 走 → 回到 x，得 42
&x;               // 取 x 的地址 → 又回到 p 存的那个值
```

## 5.3 为什么要指针：函数改不了调用方的变量（核心动机）

这是指针存在的头号理由，**整本教材的主线**。看这个「失败的交换」：

```c
void swap(int a, int b) {     // 形参 a、b 是【副本】
    int t = a; a = b; b = t;  // 换的是副本！
}
int main(void) {
    int x = 1, y = 2;
    swap(x, y);
    printf("%d %d\n", x, y);  // 还是 1 2 ❌
}
```

**C 只有值传递**：函数收到的是实参的**拷贝**，函数内改形参，跟调用方无关。

怎么解决？**别传值，传地址**——函数拿到地址，就能顺着地址去改原变量：

```c
void swap(int *a, int *b) {        // 收到的是两个地址
    int t = *a;                     // *a：跟着 a 存的地址走过去，取值
    *a = *b;                        // 把 b 指向的值写到 a 指向的地方
    *b = t;
}
int main(void) {
    int x = 1, y = 2;
    swap(&x, &y);                   // 传 x、y 的地址
    printf("%d %d\n", x, y);        // 2 1 ✅
}
```

图解（`swap(&x, &y)` 时，函数内 `a=&x`、`b=&y`）：

```
函数内：a ──► [x=1]      b ──► [y=2]
         (存x地址)            (存y地址)

int t = *a;   // t = 1
*a = *b;      // x = 2   (跟着 a 走到 x，改成 b 指向的值)
*b = t;       // y = 1
```

> **记住这条链，它会反复出现**：
> **「函数要改调用方的变量 → 传它的地址 `&x` → 函数用 `*地址` 改它」**
>
> 这条链解释了：scanf 为什么 `&`、链表头插为什么用二级指针、为什么函数能改数组（数组名本身就是地址）。

## 5.4 指针当函数返回值要小心：悬空指针

函数可以返回指针，但**绝不能返回局部变量的地址**：

```c
int *bad(void) {
    int local = 42;
    return &local;     // ❌ local 出函数就销毁，地址失效
}
```

`local` 是局部变量，函数一返回内存就被回收，返回的地址指向「已经被回收的内存」——这叫**悬空指针**，解引用它结果不可预测（可能崩、可能读到垃圾）。**这就是后面要学 `malloc` 的核心动机**：要造一个能返回出去长期用的东西，局部变量不行，得用堆。

## 5.5 指针与数组：天然一对

数组名 = **首元素地址**（一个不能改的常量指针）。

```c
int a[5] = {10, 20, 30, 40, 50};
int *p = a;            // a 退化成 &a[0]，p 指向 a[0]
```

```
        a[0]  a[1]  a[2]  a[3]  a[4]
内存:   [10]  [20]  [30]  [40]  [50]
地址:   0x100 0x104 0x108 0x10c 0x110
         ↑
         p（= a）
```

通过指针访问数组，三种等价写法：

```c
printf("%d\n", *p);       // 10  —— p 指向的元素
printf("%d\n", a[0]);     // 10  —— 下标写法
printf("%d\n", *(p+1));   // 20  —— 指针 +1 后解引用
printf("%d\n", p[1]);     // 20  —— 指针也能用下标！
```

**核心等价关系**：`p[i]` 完全等价于 `*(p + i)`，永远成立。下标 `[]` 就是指针运算的语法糖。

### 指针算术：`+1` 跨过一个元素，不是 1 字节

```c
p + 1;   // 不是地址 +1 字节，而是「跨过 p 指向的类型」
```

`int *p`，`p+1` 地址增加 `sizeof(int)=4` 字节，指向下一个 int。
`double *p`，`p+1` 增加 8 字节。
`char *p`，`p+1` 增加 1 字节。

**指针必须带类型，就是因为类型决定 `+1` 跨多大**。

```c
p++;         // p 指向 a[1]
p += 2;      // p 指向 a[3]
int d = q - p;  // 两指针相减 = 中间隔几个元素（不是字节数）
```

### 用指针遍历数组（不用下标）

```c
int sum = 0;
for (int *p = a; p < a + 5; p++)   // p 从头走到尾
    sum += *p;
// a + 5 是「尾后位置」，合法做边界，但不能解引用
```

> 这正是数组传参退化的根源：数组传给函数时退化成首元素指针，所以形参 `int a[]` 和 `int *a` 完全等价，函数内拿不到长度，要单独传 `n`。

## 5.6 指针与结构体：`.` vs `->`

结构体变量用 `.`，结构体指针用 `->`：

```c
struct Node {
    int data;
    int next;     // 先用 int 占位，下章换成指针
};

struct Node n = {42, 0};      // 变量
struct Node *p = &n;          // 指针，指向 n

n.data;        // 变量用 .
p->data;       // 指针用 ->，等价于 (*p).data
```

`p->data` 就是 `(*p).data` 的简写——先解引用拿到结构体，再取成员。因为太常用，发明了 `->` 简写。

### `p->data` 取的是「值」，不是地址（易混）

```c
struct Node { int data; int next; };
struct Node n = {42, 0};
struct Node *p = &n;

p->data        // 拿到的是 42 这个【值】，不是地址！
```

拆开：`->` = 先走过去拿结构体，再取某成员的**数据**。要地址得自己加 `&`：

| 写法 | 拿到什么 |
|------|---------|
| `p` | 结构体 n 的地址（`Node *`） |
| `*p` | 结构体 n 本身（`Node`） |
| `p->data` | data 成员的**值**（`int`） |
| `&(p->data)` | data 成员的地址（`int *`） |

> 记住：`.` 和 `->` 都是**取成员**，取出来的是成员的**值**；要地址写 `&`。

**一句记忆**：
- 拿到**变量本身**（真身）→ 用 `.`
- 拿到**地址**（指针）→ 用 `->`

### 结构体数组里的指针（你 abc.c 总结过）

```c
struct Hero team[3] = {{0,100},{1,100},{2,100}};
struct Hero *p = team;        // p 指向 team[0]

p->id;          // p 是指针，用 ->
p[0].id;        // p[0] 取出的是真身（一个 struct Hero），用 .
p[1].id;        // 同理，真身用 .
```

**为什么 `p[i]` 用 `.`？** 因为 `p[i]` = `*(p+i)`，解引用后得到的是一个**结构体变量**（真身），所以用 `.`。而 `p` 本身是指针，用 `->`。

## 5.7 const 指针（认得就行）

```c
const int *p;        // 指向 const int：不能通过 p 改数据，但 p 能指向别处
int *const p = &x;   // const 指针：p 本身不能改（不能指向别处），能改数据
const int *const p;  // 都不能改
```

数据结构代码里最常见 `const Type *`：函数承诺「我只读不写你的数据」。看到认得即可，不用记三种组合的细节。

## 5.8 指针的四大坑（必看）

1. **野指针**：指针没初始化就用。
   ```c
   int *p;          // p 是随机地址
   *p = 5;          // ❌ 往随机内存写，可能崩
   ```
   习惯：声明就初始化 `int *p = NULL;` 或马上赋有效地址。

2. **NULL 解引用**：
   ```c
   int *p = NULL;
   *p = 5;          // ❌ NULL 没有有效地址，必崩
   ```
   用前判空：`if (p != NULL) *p = 5;`。链表里 `if (p) p->data` 天天用。

3. **悬空指针**：指向已失效的内存（返回局部变量地址、free 后还用——08 章）。
   ```c
   int *p = bad();   // bad 返回局部变量地址，见 5.4
   *p;               // ❌ 那内存已失效
   ```

4. **越界指针**：
   ```c
   int a[5];
   int *p = a + 10;   // 越界，解引用崩
   ```

> 开 `-fsanitize=address` 编译能立刻抓 1、2、4，学指针时强烈建议开：
> `clang -Wall -fsanitize=address a.c -o a && ./a`

## 5.9 二级指针：存「指针的地址」（链表必用）

指针自己也是变量，也有地址。**存指针地址的变量 = 二级指针**。

```c
int x = 5;
int *p = &x;        // 一级指针：存 x 的地址
int **pp = &p;      // 二级指针：存 p 的地址
```

```
        p               x
内存:  [ &x ]  ──────► [ 5 ]
         ↑
         pp
内存:  [&p]   ──────►
```

逐层解引用：
- `pp` 的值是 `p` 的地址。
- `*pp` = `p`（一级指针，即 x 的地址）。
- `**pp` = `*p` = `x` = 5。

### 解一层 `*pp` 改地址，解两层 `**pp` 改值

这是二级指针最绕的点，记牢：指针变量的「值」本身就是个地址，所以对它赋值 = 改地址。

| 表达式 | 落在谁上 | 赋值改的是 | 改地址还是改值 |
|--------|---------|-----------|---------------|
| `*pp` | 指针 p 本身 | `*pp = &y` 让 p 改指向 | **改地址** |
| `**pp` | 数据 x | `**pp = 99` 改 x 的内容 | **改值** |

对照一级指针，规律完全一样，只是多套一层：

| | 改地址（改指向） | 改值（改数据） |
|---|---|---|
| 一级指针 `p` | `p = &y` | `*p = 99` |
| 二级指针 `pp` | `*pp = &y` | `**pp = 99` |

> 规律：**不管几级指针，"直接写指针名" = 改它存的地址（改指向）；"带 `*` 解引用到终点" = 改数据值。** 二级指针没发明新规则，只是同样的规律多套一层。

### 为什么链表要二级指针？（关键，反复想）

回到 5.3 的链：**函数改不了调用方的变量本身，只能改它指向的东西**。

- 想在函数里改一个普通 `int` → 传 `int *`（它地址），用 `*` 改。
- 想在函数里改一个**指针** `Node *head` → 传 `Node **`（指针的地址），用 `*` 改这个指针本身。

链表头插就是这个场景：函数要把调用方的 `head` 改成指向新节点。

```c
// head 本身是个 Node* 指针；要改它，得传它的地址 Node**
void insert_front(Node **head, int val) {
    Node *node = malloc(sizeof(Node));   // 造新节点，08 章讲
    node->next = *head;    // *head 就是调用方那个 head 指针指向的节点
    *head = node;          // 把调用方的 head 改成指向新节点！
}
// 调用：
Node *head = NULL;
insert_front(&head, 5);   // 传 head 的地址
```

如果只用一级指针 `void f(Node *head)`，函数里 `head = node` 只改了副本，调用方的 head 不变——和 5.3 失败的 swap 一模一样的毛病。

> 另一种更简单的写法（10 章主用）：让函数**返回新 head**，调用方 `head = insert_front(head, 5);`。避开二级指针，但本质是同件事——把「改 head」这件事挪到调用方做。两种写法都认得。

### 一道想通题

为什么这个改不了调用方的指针？
```c
void set_null(int *p) { p = NULL; }   // 调用 set_null(ptr) 后 ptr 还是原值
```
答：`p` 是 `ptr` 的**副本**，把副本置 NULL 不影响原指针。要真改原指针，得传它的地址：`void set_null(int **p) { *p = NULL; }`，调用 `set_null(&ptr)`。

**这条想通，二级指针就通了。**

### 补充：`int **` 还能搭「指针数组式」二维（跟 `int m[][4]` 区分）

`int **` 除了改指针，还能当一种**二维结构**用：先有个行表（指针数组），每行一个独立指针，各行长度可不等。

```c
int row0[4] = {1,2,3,4}, row1[3] = {5,6,7};
int *rows[2] = {row0, row1};   // 行表：每行一个 int*
int **p = rows;                // p 指向行表
p[1][2];   // = 7。两步：p[1] 取第1行指针，再 [2] 访问行内
```

跟 04 章的真·二维数组对比：

| | 真·二维 `int m[3][4]` | 指针数组 `int **p` |
|---|---|---|
| 访问 | 一个起点 + 固定公式 `i*cols+j` | 先取行指针，再访问行内 |
| 行长 | **必须等长**（公式里 cols 写死） | **可不等长**（各行独立指针） |
| 内存 | 一整块连续 | 行分散，行表另存 |

> 这种"指针数组式"二维真正大展拳脚是在动态内存（08 章）：行数列数运行时才知道，就先 `malloc` 行表再给每行 `malloc`。注意它**不是**数组名退化（`int m[][4]` 退化成 `int (*m)[4]`，跟 `int **` 不能互传）。

## 易错点

1. `int *p` 声明的 `*` 是类型标记，不是解引用；`*p` 表达式才是解引用。
2. `p + 1` 跨过一个元素（按类型大小），不是 1 字节。
3. `p[i]` ≡ `*(p+i)`，永远成立；下标是指针运算的语法糖。
4. 结构体指针用 `->`，结构体变量用 `.`；`p[i]` 解出真身用 `.`。
5. 改调用方**普通变量**用一级指针 `*`；改调用方**指针本身**用二级指针 `**`。
6. 指针声明就初始化（最好 NULL），用前判空。
7. 别返回局部变量的地址（悬空）。

## 例题

**例 5.1** 下面输出？
```c
int x = 10, y = 20;
int *p = &x, *q = &y;
*p = *q;          // 把 q 指向的值赋给 p 指向的地方
printf("%d %d\n", x, y);
```
解析：`*p` 是 x，`*q` 是 y。`*p = *q` 即 `x = y`。
答：`20 20`。注意和 `p = q`（改指针指向）的区别。

**例 5.2** `p = q` vs `*p = *q` 的区别。
```c
// 假设 p=&x, q=&y
*p = *q;   // x = y：改的是数据，p 仍指向 x，q 仍指向 y
p = q;     // p 也指向 y 了：改的是指针的指向，x 没被碰
```
这俩是初学者最易混的，画图区分。

**例 5.3** 下面输出？
```c
int a[3] = {1, 2, 3};
int *p = a;
printf("%d %d %d\n", *p, *(p+2), p[1]);
```
答：`1 3 2`。`*p`=a[0]=1，`*(p+2)`=a[2]=3，`p[1]`=a[1]=2。

**例 5.4** 下面输出？
```c
struct S { int x; } s = {7};
struct S *p = &s;
p->x = 10;
printf("%d\n", s.x);
```
答：`10`。`p->x` 改的就是 `s.x`。

**例 5.5** 用指针遍历数组求和（不下标）。
```c
int sum = 0;
for (int *p = a; p < a + n; p++) sum += *p;
```

**例 5.6** 下面输出？
```c
int x = 5;
int *p = &x;
int **pp = &p;
**pp = 99;
printf("%d\n", x);
```
答：`99`。`**pp` → `*p` → `x`，改的就是 x。

## 小练习

1. 写 `void swap(int *a, int *b)` 交换两数，并写 main 验证。
2. 写 `void add_to_all(int *a, int n, int k)`，用指针把数组每个元素加 k（不下标）。
3. 写 `int *max_ptr(int *a, int n)`，返回数组中最大元素的地址；调用方用它把最大元素改成 0。
4. 写 `void double_it(int *p)` 把 `*p` 翻倍；再写 `void double_ptr(int **pp)`，它把 `*pp`（一个指针）改成 NULL。理解两者的区别。
5. 画图解释：`*p = *q` 和 `p = q` 跟着两个不同变量的关系。

<details><summary>答案</summary>

```c
// 1
void swap(int *a, int *b) { int t = *a; *a = *b; *b = t; }
int main(void) {
    int x = 1, y = 2;
    swap(&x, &y);             // 传地址
    printf("%d %d\n", x, y);  // 2 1
}

// 2
void add_to_all(int *a, int n, int k) {
    for (int *p = a; p < a + n; p++) *p += k;
}

// 3
int *max_ptr(int *a, int n) {
    int *m = a;
    for (int i = 1; i < n; i++)
        if (a[i] > *m) m = &a[i];
    return m;
}
// 调用：int *p = max_ptr(a, n); *p = 0;

// 4
void double_it(int *p) { *p *= 2; }            // 改 p 指向的 int
void double_ptr(int **pp) { *pp = NULL; }      // 改调用方那个指针本身
// 区别：double_it 改数据，用一级指针；
//       double_ptr 改指针，用二级指针（因为指针本身也是要被改的"变量"）

// 5
// 假设 p=&x, q=&y：
//   *p = *q：x = y。p 还指着 x，q 还指着 y。数据变了，指向没变。
//   p = q：p 也指向 y 了。x、y 数据没变，只是 p 不再指 x。
```
练习 4、5 是区分「改数据」和「改指向」的关键，务必想通——它直接决定你能不能看懂链表代码。

指针的核心就两句话：
1. **指针是存地址的变量；`*` 解引用、`&` 取地址，互逆。**
2. **函数要改调用方的变量，就传它的地址。**（普通变量传 `&x`→`int*`；指针本身传 `&p`→`int**`。）
