# 06 结构体

结构体 = 把几个相关变量**打包成一个整体**。链表节点、树节点、图边都是结构体。这章是 09 章自引用结构体的前奏，要扎实。

## 6.1 定义结构体

```c
struct Student {
    char name[20];
    int  age;
    double gpa;
};
```

声明变量：
```c
struct Student s1;                 // 注意要写 struct
struct Student s2 = {"Alice", 20, 3.8};
```

访问成员用 `.`：
```c
s1.age = 21;
strcpy(s1.name, "Bob");
```

## 6.2 typedef：起别名，省 `struct`

`struct Student` 每次都要写 `struct` 很烦。用 `typedef` 起别名：

```c
typedef struct Student {
    char name[20];
    int  age;
} Student;          // 现在 Student 就是 struct Student 的别名

Student s = {"Alice", 20};   // 不用写 struct 了
```

更常见、更简洁的写法（数据结构教材几乎都这样）：

```c
typedef struct {
    char name[20];
    int  age;
} Student;
```

> 408 数据结构代码里节点定义基本都是 `typedef struct { ... } Node;` 形式，习惯它。

## 6.3 结构体指针与 `->`

```c
Student s = {"Alice", 20};
Student *p = &s;

p->age = 21;          // 指针用 ->，等价 (*p).age
strcpy(p->name, "A");
```

复习（05 章）：变量用 `.`，指针用 `->`。

## 6.4 结构体数组

```c
typedef struct {
    int id;
    int score;
} Player;

Player team[100];
for (int i = 0; i < n; i++)
    scanf("%d %d", &team[i].id, &team[i].score);
```

访问第 i 个的成员：`team[i].id`。取第 i 个的地址传给函数：`&team[i]`。

## 6.5 结构体作函数参数

```c
// 值传递：整个结构体被拷贝一份（小结构体无所谓）
void print(Player p) { printf("%d %d\n", p.id, p.score); }

// 传指针：不拷贝，能改原结构体（大结构体或要修改时用）
void add_score(Player *p, int bonus) { p->score += bonus; }

// 加 const：传指针但禁止修改，防误改（只读访问推荐）
void show(const Player *p) { printf("%d\n", p->score); }
```

> 数据结构里节点很大时（含数组/多个字段）一律传指针 `Node *`，别值传拷贝。

## 6.6 结构体整体赋值

和数组不同，结构体可以**整体赋值**（深拷贝字段，包括里面的数组）：

```c
Player a = {1, 90};
Player b = a;          // b 是 a 的副本，改 b 不影响 a
```

这给链表「节点复制」带来方便，但也别误以为它复制了指针指向的东西——**指针成员只复制地址，不复制指向的内存**（这就是浅拷贝，动态内存时要小心）。

## 易错点

1. 忘 `typedef` 导致每次写 `struct`。
2. 结构体指针用 `.`（应该 `->`）。
3. 字符串成员赋值不能用 `=`：`s.name = "Bob";` 非法（数组不能赋值），要用 `strcpy(s.name, "Bob");`。
4. 大结构体值传递性能差，传指针。
5. `typedef struct {...} Student;` 末尾分号别漏。

## 例题

**例 6.1** 定义点结构体，写函数算两点距离。
```c
typedef struct { double x, y; } Point;

double dist(Point a, Point b) {
    double dx = a.x - b.x, dy = a.y - b.y;
    return sqrt(dx*dx + dy*dy);    // #include <math.h>，编译加 -lm
}
```

**例 6.2** 结构体数组按 score 降序排序（你做过，回顾稳定排序思路）。
```c
// 比较：score 大的在前，相同则保持原序
for (int i = 0; i < n - 1; i++)
    for (int j = 0; j < n - 1 - i; j++)
        if (team[j].score < team[j+1].score) {
            Player t = team[j]; team[j] = team[j+1]; team[j+1] = t;
        }
```

**例 6.3** 下面输出？
```c
typedef struct { int a; } S;
void f(S s) { s.a = 99; }          // 值传递
int main(void) {
    S x = {1}; f(x);
    printf("%d\n", x.a);           // 1，没改到原结构体
}
```

## 小练习

1. 定义 `typedef struct { int h, m, s; } Time;`，写 `void add_one_sec(Time *t)`，处理秒数进位（59→00，分+1；分类推）。
2. 写函数 `Student *find_by_id(Student *arr, int n, int id)`，返回匹配元素的指针，没有返回 NULL。
3. 解释例 6.3 改成 `void f(S *s)` 后会怎样。

<details><summary>答案</summary>

```c
// 1
void add_one_sec(Time *t) {
    t->s++;
    if (t->s == 60) { t->s = 0; t->m++;
        if (t->m == 60) { t->m = 0; t->h++;
            if (t->h == 24) t->h = 0;
        }
    }
}

// 2
Student *find_by_id(Student *arr, int n, int id) {
    for (int i = 0; i < n; i++)
        if (arr[i].id == id) return &arr[i];
    return NULL;
}
// 调用：Student *p = find_by_id(team, n, 5); if (p) p->score++;

// 3 改成 void f(S *s) { s->a = 99; } 后，调用 f(&x)，x.a 变成 99。
//    因为传的是地址，能改原结构体。
```
