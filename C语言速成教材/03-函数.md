# 03 函数

函数 = 把一段代码打包，取个名字，重复用。数据结构里**所有操作都写成函数**（建表、插入、删除……），所以这章要扎实。

## 3.1 定义与调用

```c
// 返回类型  函数名(参数列表)
int add(int a, int b) {
    return a + b;     // return 把结果送回调用方，并结束函数
}

int main(void) {
    int s = add(3, 5);   // 调用，s = 8
    return 0;
}
```

- `int`：返回类型。不返回值用 `void`。
- `(int a, int b)`：形参（形式参数），是函数内部的局部变量，调用时用实参初始化。
- `return`：送回值并退出。`void` 函数可 `return;` 提前退出。

### 函数的分类

函数分两个维度，组合起来就是四种类型：

**维度一：有没有返回值**
- 有返回值：`int / double / char / 指针 …`，**必须有** `return 值;`
- 无返回值：`void`，只做事不交结果，`return;` 可选（提前退出用）

**维度二：有没有参数**
- 有参：`(int a, int b)`，调用时要给值
- 无参：`(void)` 或空 `()`，调用时不给值（建议写 `(void)`，空 `()` 在老 C 里含义模糊）

四种典型组合：

```c
// ① 有返回值 + 有参数（最常见，算个结果交回去）
int add(int a, int b) { return a + b; }

// ② 有返回值 + 无参数（如读个状态、取随机数）
int get_count(void) { return counter; }

// ③ 无返回值 + 有参数（对参数做点事，不交结果）
void print_line(int n) {
    for (int i = 0; i < n; i++) putchar('-');
    putchar('\n');
}

// ④ 无返回值 + 无参数（纯粹的副作用，如初始化）
void greet(void) { printf("hello\n"); }
```

> 数据结构里几乎都是**①或③**：`int length(List)` 这种是①（返回长度）；`void insert(List*, int x)` 这种是③（往表里插，不返回值，通过指针参数把结果"带"出去）。

**一句话判断该用哪种**：要"算出一个值给调用方" → 有返回值；只是"改点东西/打印点东西" → `void`。

## 3.2 参数传递：C 只有「值传递」

**这是 C 最重要的一条规则，记住**：函数收到的是实参的**副本**。函数内改形参，**不影响调用方的原变量**。

先把三个词分清：

| 名词 | 是什么 | 在哪 |
|------|--------|------|
| **实参**（实际参数） | 调用时你给的具体值，如 `swap(x, y)` 里的 `x`、`y` | 调用方 |
| **形参**（形式参数） | 定义时括号里的占位变量，如 `void swap(int a, int b)` 里的 `a`、`b` | 函数内部 |
| **副本** | 实参的拷贝，赋给形参 | 函数内部 |

**调用瞬间发生的事**：编译器拿实参的值，**复制一份**塞进形参这个局部变量里。之后形参和实参就是两个独立的盒子，函数怎么折腾形参都不碰原变量。

```c
void swap(int a, int b) {        // a、b 是 x、y 的副本
    int t = a; a = b; b = t;     // 改的是副本！原变量没动
}
int main(void) {
    int x = 1, y = 2;
    swap(x, y);                  // 把 1、2 复制给 a、b
    printf("%d %d\n", x, y);     // 仍是 1 2 ❌ 没换成功
}
```

**为什么没换成功**：`swap` 里确实把 `a`、`b` 互换了，但 `a`、`b` 只是 `x`、`y` 的拷贝，函数一返回 `a`、`b` 就销毁了，`x`、`y` 压根没被碰过。

### 传指针 = 还是值传递，只是传的是「地址的值」

```c
void swap(int *a, int *b) {
    int t = *a; *a = *b; *b = t;   // 通过地址改原变量
}
int main(void) {
    int x = 1, y = 2;
    swap(&x, &y);                  // 传地址
    printf("%d %d\n", x, y);       // 2 1 ✅
}
```

这里 `swap` 收到的 `a`、`b` 仍然是**副本**——只不过是"地址的副本"。`&x` 这个地址值被复制给形参 `a`，于是 `a` 和 `&x` 指向同一块内存。函数里 `*a` 解引用，改的就是 `x` 本身。所以：

> **本质没变，还是值传递。** 只不过传的值是"地址"，有了地址就能顺藤摸瓜改到原变量。

### 三种典型传参场景对照

```c
// ① 传值：函数只要"读"这个值，不改它
int square(int n) { return n * n; }       // 调用 square(5)

// ② 传指针：函数要"改"调用方的变量
void set_zero(int *p) { *p = 0; }          // 调用 set_zero(&a)

// ③ 传数组：数组名自动退化成首地址（见 04 章）
int sum(int arr[], int n) { /* 求和 */ }   // 调用 sum(nums, 10)
```

> **记住这条链**：函数改不了调用方的变量本身 → 传它的地址 → 函数用 `*地址` 改它。链表里「在函数里改头指针」靠这条，用二级指针（05 章、10 章会用到）。

数组作参数也一样——传的是数组首地址的副本（见 04 章「退化」）。

## 3.3 函数声明（原型）

如果函数定义在调用之后，或跨文件，要先**声明**（告诉编译器有这么个函数）：

```c
int add(int a, int b);   // 声明，分号结尾，可省形参名

int main(void) {
    printf("%d\n", add(3, 5));
    return 0;
}

int add(int a, int b) { return a + b; }   // 定义
```

习惯：把声明放头文件或文件开头，定义放后面。`#include <stdio.h>` 干的就是这事——把一批声明塞进来。

## 3.4 局部变量与全局变量

- **局部变量**：函数内声明。进函数创建、出函数销毁。**函数之间互不可见**。
- **全局变量**：函数外声明，整个程序可见、全程存活。
- `static` 局部变量：只初始化一次，函数退出后仍在，但作用域还是局部。

```c
int counter = 0;          // 全局

void count(void) {
    counter++;            // 多次调用累加
}
```

> 全局变量方便但要克制用——容易在不知哪里被改。数据结构里「头指针」一般放 main 局部变量，传给函数，而不是用全局。

## 3.5 返回局部变量的指针？不行！

```c
int *bad(void) {
    int x = 5;
    return &x;    // ❌ x 出函数就没了，返回的是悬空指针
}
```
出函数后 `x` 销毁，返回的地址失效。**这是引入动态内存的核心动机**（08 章详讲）。

## 易错点

1. 以为函数能改调用方的普通变量——不能，值传递。要改就传指针。
2. 忘声明导致编译器猜错返回类型（默认 int，会出诡异 bug）。
3. 返回局部变量地址——悬空指针。
4. `void` 函数里 `return 值;` 是错的。
5. 形参和实参类型不匹配（如传 `int` 给 `double` 形参）会隐式转换，可能丢精度。

## 例题

**例 3.1** 写函数 `int max3(int a, int b, int c)` 返回三者最大。
```c
int max3(int a, int b, int c) {
    int m = a;
    if (b > m) m = b;
    if (c > m) m = c;
    return m;
}
```

**例 3.2** 下面输出？
```c
void f(int x) { x = 10; }
int main(void) {
    int a = 0; f(a);
    printf("%d\n", a);   // 0，值传递，改的是副本
}
```
答：`0`。

**例 3.3** 用指针写 `void double_it(int *p)`，把 `*p` 翻倍。
```c
void double_it(int *p) {
    *p = (*p) * 2;   // 或 *p *= 2
}
```

## 小练习

1. 写 `int is_even(int n)` 判断偶数；`int gcd(int a, int b)` 用辗转相除法求最大公约数。
2. 写 `void minmax(int a, int b, int *pmin, int *pmax)`，用一次调用同时得到最小最大值。
3. 改错：返回局部数组地址的函数。

<details><summary>答案</summary>

```c
// 1
int is_even(int n) { return n % 2 == 0; }
int gcd(int a, int b) {
    while (b != 0) { int t = a % b; a = b; b = t; }
    return a;
}

// 2
void minmax(int a, int b, int *pmin, int *pmax) {
    if (a < b) { *pmin = a; *pmax = b; }
    else       { *pmin = b; *pmax = a; }
}
// 调用：int lo, hi; minmax(3, 8, &lo, &hi);   —— 一个函数返回多个值的常用招

// 3 错误示例：
int *bad(void) {
    int arr[3] = {1,2,3};
    return arr;          // ❌ 局部数组，出函数销毁
}
// 数据结构里要"造一个东西返回出去给别人用"，靠 malloc（08 章）
```
