# 12 递归

递归 = 函数**调用自己**。二叉树遍历、分治、回溯都靠它。你的代码里没出现过递归，这章建立直觉。

## 12.1 递归的两要素

每个递归函数必须有：
1. **基线条件（base case）**：不再递归、直接返回的简单情况。**没有它就无限递归→栈溢出崩溃**。
2. **递归条件**：把问题缩小一点，调用自己解决更小的同类问题。

```c
// 阶乘：n! = n * (n-1)!
int fact(int n) {
    if (n <= 1) return 1;     // 基线：0! = 1! = 1
    return n * fact(n - 1);   // 递归：n! = n * (n-1)!
}
```

执行 `fact(4)`：
```
fact(4) = 4 * fact(3)
             = 4 * 3 * fact(2)
                  = 4 * 3 * 2 * fact(1)
                              = 4 * 3 * 2 * 1 = 24
```

**心法**：递归不是「循环」，而是「把大问题分解成结构相同的小问题，直到小到能直接答」。相信递归函数对更小输入是正确的（递归信仰之跃），别去一步步展开。

## 12.2 递归的执行：调用栈

每次调用自己，系统在「调用栈」上压一层（保存参数、局部变量、返回地址）。基线条件触底后，逐层弹栈返回。

- 递归太深（如 `fact(1000000)`）会**栈溢出**（Stack Overflow）——栈有大小限制（通常几 MB）。
- 这就是为什么有些递归要改成循环（如求和、求长度）。

## 12.3 经典例子

### 斐波那契（递归直译，但慢）
```c
int fib(int n) {
    if (n < 2) return n;          // fib(0)=0, fib(1)=1
    return fib(n-1) + fib(n-2);   // 递归
}
```
> 这个版本 O(2ⁿ) 慢得离谱（重复计算），但**最直观展示递归思想**。408 考递归思想用它，工程上用循环或记忆化。

### 幂
```c
double power(double x, int n) {
    if (n == 0) return 1;
    return x * power(x, n - 1);
}
```

### 求和（递归版，演示用）
```c
int sum(int n) {            // 1+2+...+n
    if (n == 0) return 0;
    return n + sum(n - 1);
}
// 工程上用循环，这里只为理解递归
```

## 12.4 递归与链表（重点：为树做准备）

链表天然递归——「一个链表 = 一个头节点 + 一个更短的链表（head->next 起的部分）」。

```c
// 递归求链表长度
int length(Node *head) {
    if (head == NULL) return 0;          // 基线：空表长度 0
    return 1 + length(head->next);       // 1 + 后面那段的长度
}

// 递归求和
int sum_list(Node *head) {
    if (head == NULL) return 0;
    return head->data + sum_list(head->next);
}

// 递归释放
void free_list(Node *head) {
    if (head == NULL) return;
    free_list(head->next);   // 先释放后面的
    free(head);              // 再释放自己
}
```

> 这正是树的写法的预演——树是「一个根 + 左子树 + 右子树」，结构完全一样。把链表递归想通，树就好办。

### 递归反转链表（进阶）

```c
Node *reverse(Node *head) {
    if (head == NULL || head->next == NULL) return head;  // 基线：空或单节点
    Node *new_head = reverse(head->next);   // 反转后面的，得到新头
    head->next->next = head;                // 原来的第二个节点指回 head
    head->next = NULL;                      // head 现在是尾
    return new_head;
}
```
这个有点烧脑，画图体会。10 章给了迭代版，两种都要会。

## 12.5 二叉树节点：递归的真正战场（预告）

树是递归的天然主场。一个树的节点定义（自引用结构体的进阶）：

```c
typedef struct TreeNode {
    int data;
    struct TreeNode *left;     // 左子树
    struct TreeNode *right;    // 右子树
} TreeNode;
```

和链表节点一模一样的套路，只是有两个 `next`（叫 left/right）。遍历用递归：

```c
// 中序遍历：左 → 根 → 右
void inorder(TreeNode *root) {
    if (root == NULL) return;          // 基线
    inorder(root->left);               // 递归左子树
    printf("%d ", root->data);         // 访问根
    inorder(root->right);              // 递归右子树
}
```

> 你看，结构和你 12.4 写的链表递归**完全同构**：基线判空 + 递归处理子结构。学好这章的链表递归，到树就是换个名字。

## 易错点

1. **忘基线条件** → 无限递归 → 栈溢出。
2. 基线条件写得不对（如 `fact` 里 `if (n == 1)` 但传入 0 会无限递归）——用 `n <= 1` 更稳。
3. 递归没让问题变小（`fact(n)` 里调用 `fact(n)` 而非 `fact(n-1)`）→ 死循环。
4. 递归太深栈溢出——大数据改循环。
5. 重复计算（朴素 fib）——理解思想即可，工程记忆化。

## 例题

**例 12.1** 递归求数组元素和 `a[0..n-1]`。
```c
int arr_sum(int a[], int n) {
    if (n == 0) return 0;
    return a[n-1] + arr_sum(a, n-1);
}
```

**例 12.2** 递归判断回文（不看数组版，看递归版）。
```c
int is_pal(const char *s, int left, int right) {
    if (left >= right) return 1;            // 基线：指针相遇或交叉
    if (s[left] != s[right]) return 0;
    return is_pal(s, left+1, right-1);       // 缩小范围
}
```

**例 12.3** 下面输出？
```c
void f(int n) {
    if (n == 0) return;
    printf("%d ", n);
    f(n - 1);
}
f(3);
```
答：`3 2 1 `（先打印再递归）。如果把两行调换（先 `f(n-1)` 再打印）则 `1 2 3 `——这是「前序 vs 后序」的本质，树的遍历会用到。

## 小练习

1. 递归求 `x^n`（n 非负整数）。
2. 递归求数组最大值。
3. 递归统计链表中有多少个节点 data 为偶数。
4. 用递归写二叉树的前序遍历（根→左→右）。

<details><summary>答案</summary>

```c
// 1
double power(double x, int n) {
    if (n == 0) return 1;
    return x * power(x, n - 1);
}

// 2
int arr_max(int a[], int n) {
    if (n == 1) return a[0];
    int m = arr_max(a, n - 1);
    return a[n-1] > m ? a[n-1] : m;
}

// 3
int count_even(Node *head) {
    if (!head) return 0;
    int rest = count_even(head->next);
    return (head->data % 2 == 0 ? 1 : 0) + rest;
}

// 4 前序：根 → 左 → 右
void preorder(TreeNode *root) {
    if (!root) return;
    printf("%d ", root->data);
    preorder(root->left);
    preorder(root->right);
}
```
练习 3、4 是递归处理「自引用结构体」的标准范式，反复练。
